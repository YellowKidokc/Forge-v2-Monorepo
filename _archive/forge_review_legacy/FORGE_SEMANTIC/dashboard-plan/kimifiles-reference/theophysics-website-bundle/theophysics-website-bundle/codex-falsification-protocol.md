# FALSIFICATION PROTOCOL — Pre-Registration Template
## Source: Codex CLI (March 7, 2026)

### Goal
Try to kill the theory, not defend it.

### Primary Claim To Test
Cross-domain coupling exists: changes in "spiritual state" produce measurable physical effects beyond standard explanations.

### Protocol

**1. Pre-register everything**
- Hypotheses, metrics, analysis, exclusions, sample size, stopping rules.
- Lock equations/constants before data collection.

**2. Define 3 killer predictions (must be unique)**
- Predictions that standard models do NOT already expect.
- Format: "If X spiritual intervention occurs, Y physical signal shifts by Z within T window."

**3. Multi-lab blinded replication**
- At least 3 independent labs.
- Double-blind condition labels.
- Shared protocol, separate data collection.

**4. Negative controls**
- Sham interventions.
- Randomized fake event windows.
- Synthetic data injection checks.

**5. Adversarial analysis**
- Independent red-team statisticians try to break results.
- Correct for multiple comparisons.
- Report all nulls.

**6. Out-of-sample test**
- Train rules on Dataset A, test unchanged on Dataset B.
- No post-hoc parameter retuning.

**7. Pass/fail thresholds**
- Primary endpoints replicate in all labs or at least 2/3 with same direction and effect size bounds.
- Predefined minimum effect size and Bayes factor / p-value criteria.
- If not met: theory fails this round.

### What Would Disconfirm Strongly
- Effects disappear under blinding.
- Results only appear after model tuning.
- Controls show same "signal" as treatment.
- Different labs can't reproduce direction/magnitude.
